# 双周期基金观察评分设计

## 背景与目标

现有 `observation-v1` 将技术、估值代理和事件分析组合成单一观察分。其中净值和指数的一年价格分位都偏向均值回归：长期上涨资产不断接近新高时，会被重复判定为位置偏高。这既不能区分长期持有条件与当前追加时机，也会把正常上涨趋势误当成风险。

本阶段升级为 `observation-v2`，将结果拆成两个互不混合的分数：

- 长期持有分：衡量底层指数或基金是否具备相对稳定的长期持有条件。
- 当前投入时机分：衡量当前价格相对趋势是否过热、走弱或正在企稳。

两项分数是研究观察指标，不表示预期收益、上涨概率或买卖建议。系统继续定位为个人基金研究辅助工具。

## 产品语义

系统不再生成一个混合总分。日报、图片和历史记录并列展示两项分数及其数据状态。

| 长期持有分 | 当前投入时机分 | 解释 |
| --- | --- | --- |
| 高 | 高 | 长期条件与当前时机均较强 |
| 高 | 低 | 长期条件较强，但当前偏热或尚未企稳 |
| 低 | 高 | 可能存在短期修复，长期依据仍偏弱 |
| 低 | 低 | 两个观察维度均偏弱 |

分数统一映射为以下非操作性等级：

- `80-100`：条件较强
- `60-79`：条件偏强
- `40-59`：条件中性
- `20-39`：条件偏弱
- `0-19`：条件较弱
- 核心数据不足：暂不可评估

## 长期持有分

长期持有分由四个可复算的规则因子组成：

| 因子 | 权重 | 主要输入 |
| --- | ---: | --- |
| 真实指数估值 | 40% | PE/PB 当前值及历史分位 |
| 长期趋势 | 30% | MA200 斜率、近 6 个月和 12 个月趋势 |
| 风险稳定性 | 20% | 近 3 年最大回撤、年化波动率 |
| 跟踪质量 | 10% | 基金净值与人民币口径基准收益的一致程度 |

估值因子使用真实估值指标，不再使用指数价格分位或基金净值分位冒充估值。估值越低并不机械等于越好：历史分位进入分段映射，极端估值获得较低分，中低分位获得较高分，并在报告中展示指标名称、当前值、历史分位和数据日期。

长期趋势因子奖励持续向上的长期趋势。MA200 正斜率和稳定的 6/12 个月正收益提高分数，因此正常创新高不会被扣分。风险稳定性用于区分平稳复利与高波动上涨。跟踪质量用于识别基金净值明显偏离对应基准的情况。

真实估值是长期持有分的核心输入。当前基金没有可验证的指数估值数据、估值缓存超过 7 个自然日或历史估值样本不足时，不生成长期持有分，并列出具体原因。系统不得回退到价格分位拼出数值。

## 当前投入时机分

当前投入时机分只使用现有净值和市场时间序列，避免增加网络和服务器压力：

| 因子 | 权重 | 主要输入 |
| --- | ---: | --- |
| 趋势状态 | 30% | 当前值相对 MA200、MA200 斜率 |
| 趋势偏离度 | 30% | 当前值相对 MA60/MA200 的波动率标准化距离 |
| 回撤与企稳 | 25% | 当前回撤、MA20 斜率、近 5/20 日变化 |
| 短期温度 | 15% | RSI14、近 5/20 日收益 |

趋势状态用于识别长期上行、震荡和长期下行。处于长期上升趋势不会因价格创新高被处罚。

趋势偏离度使用滚动波动率标准化，而不是固定百分比或一年价格分位。只有短时间显著偏离趋势时才判定为过热；不同波动特征的指数使用自身历史尺度。

回撤本身不直接产生高分。只有在发生回撤且 MA20 转平或向上、短期跌势减弱时，企稳因子才提高。持续下跌状态即使回撤很深，也不能获得高时机分。

短期温度作为小权重辅助因子，避免 RSI 单独主导结果。系统删除原净值一年分位、指数一年价格分位以及“低于 MA60 即高分”的机械规则。

## 事件分析

LLM 事件分析不参与两项数字评分。模型只总结已经提供的新闻和结构化市场事实，并显示模型、生成时间、证据和分析状态。

原因是模型输出无法像确定性指标一样稳定复算。将其与规则评分分离，可以避免提示词、模型版本或新闻措辞变化造成数字漂移。重大事件仍在报告顶部显示风险提示，但不伪装成量化因子。

## 估值数据获取与缓存

新增估值提供器边界，输入为基准标识，输出统一结构：

- `benchmark_code`
- `metric`，例如 `pe_ttm` 或 `pb`
- `value`
- `percentile`
- `data_date`
- `source`
- `fetched_at`

A 股指数优先使用当前 AKShare 已提供的中证指数或指数市盈率接口。美股指数只接入经过样本验证、可以直接返回指数估值的轻量来源；个股估值接口不得假定为指数估值接口。

估值结果写入 SQLite。每个基准、指标和数据日期唯一；同一自然日内的多只基金共享缓存。正常每日任务每个基准最多发起一次估值请求，不增加 LLM 调用。

请求失败时：

1. 保留失败状态和错误摘要。
2. 读取最近一次有效缓存。
3. 缓存不超过 7 个自然日时允许使用，并明确展示真实数据日期和“缓存”状态。
4. 缓存超过 7 日或从未成功获取时，长期持有分标记为暂不可评估。

历史分位必须来自同一指标和同一来源。样本期、样本数和最早日期进入评分审计信息，防止不同口径混用。

## 数据流与质量状态

```text
刷新净值与市场数据
  -> 按基准读取或刷新每日估值缓存
  -> 分别检查净值、基准、估值、汇率、新闻状态
  -> 计算长期持有分或记录不可评估原因
  -> 计算当前投入时机分或记录不可评估原因
  -> 附加独立 AI 事件说明
  -> 保存 observation-v2 审计快照
  -> 更新到期结果评价
  -> 生成日报、图片和周报
```

数据质量不再只显示笼统的“市场未知”。每项输入分别报告 `ok`、`cached`、`stale`、`missing` 或 `failed`，并包含数据日期。QDII 至少区分纳指行情、USD/CNY、纳指估值和新闻刷新状态。

当前投入时机分需要足够的净值或基准序列计算 MA200 和波动率；样本不足时不评分。长期持有分除时间序列外还要求有效估值。一个分数不可评估不阻止另一个分数展示。

## 存储与兼容

旧 `observation-v1` 记录保持不变。`observation-v2` 审计快照增加：

- `long_term_score`、`long_term_level`、`long_term_json`
- `timing_score`、`timing_level`、`timing_json`
- 各输入的数据质量与日期
- `scoring_version = observation-v2`

新增估值缓存表，以基准、指标和日期为唯一键。迁移必须幂等，不删除旧字段和历史记录。旧观察分只在历史页面中按原语义展示，不能与新双评分直接比较或连成同一趋势线。

结果评价分别按 `long_term_score` 和 `timing_score` 分组统计 5、20、60 个基金交易日后的超额收益。样本少于 30 条继续标记证据不足，不根据少量样本自动调权。

## 资源约束

实现只使用现有 Python、pandas、numpy 和 SQLite：

- 不增加常驻服务、任务队列、机器学习模型或重型依赖。
- 时间序列最多读取约 3 年，单只基金计算复杂度为 `O(n)`。
- 约 5 只基金共享市场与估值数据，不重复请求。
- 每日新增开销主要是一条轻量估值 HTTP 请求和一次 SQLite upsert。
- 不增加 AI 调用次数，AI 事件分析从评分链路移到说明链路。

该方案适用于 1 核 1 GB 服务器，主要风险是外部估值源稳定性，而不是 CPU 或内存消耗。

## 错误处理

- 单个估值源失败不阻断净值刷新、时机分计算或其他基金。
- 数据源返回空表、字段变化、非数值或异常日期时视为失败，不写入有效缓存。
- 估值缓存写入失败时记录错误，不用内存中的未持久化数据生成不可审计的长期分。
- 数据库迁移失败时终止任务，避免写入未知结构。
- 单个分数计算异常只影响该分数，并保留结构化原因。
- 报告图片失败时继续降级为文字推送。

## 报告设计

日报首先展示一句自然语言摘要，再展示两项分数：

```text
长期持有 72/100 · 条件偏强
当前投入时机 38/100 · 条件偏弱

长期趋势保持向上，但当前偏离中期趋势较高，暂未出现明显回撤企稳。
```

每项分数下最多展示三个主要驱动因素及数据日期。估值不可用时必须明确说明，不用默认 `50` 或估值代理替代。报告继续保留非投资建议声明，并避免“买入”“加仓”“抄底”等指令性措辞。

## 测试与验收

所有网络请求使用替身数据，新增以下确定性场景：

- 稳定上升且不断创新高：长期趋势得分较高，时机分不因一年高分位下降。
- 上升趋势中突然大幅偏离均线：长期趋势保持，时机分下降。
- 深度回撤且跌势仍在加速：时机分不得进入条件偏强。
- 深度回撤后 MA20 转平或向上：时机分高于未企稳场景。
- 真实估值处于合理历史分位：长期估值因子提高。
- 估值缺失或过期：长期持有分不可评估，时机分仍可生成。
- 同一基准的多只基金：每日只调用一次估值提供器。
- 估值请求失败：7 日内缓存可用，过期缓存不可用于评分。
- 数据库迁移重复执行且不破坏 `observation-v1` 记录。
- 日报、周报和图片展示双评分、精确数据状态且不含操作指令。

完成标准：

1. 全部 `pytest` 测试通过。
2. `python -m compileall -q src scripts` 通过。
3. 使用固定样本生成双评分日报 Markdown 和 Dashboard 图片。
4. 人工检查桌面和移动宽度下图片文字无重叠、无截断。
5. README、配置示例和评分版本说明与实际行为一致。
