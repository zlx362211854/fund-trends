# 可信基金观察工具设计

## 背景与目标

当前系统可以完成基金数据抓取、三维评分、日报/周报生成和微信推送，但评分语义、异常降级和结果验证仍不足以支撑专业的个人研究辅助场景。

本阶段将系统定位为“专业的个人基金研究辅助工具”，而不是投资顾问或买卖建议系统。目标是让每个输出都能回答三个问题：数据是否可靠、分数依据是什么、历史样本是否支持该信号。

本阶段不接入真实 PE/PB 或持仓加权估值，不根据有限样本调整评分权重，也不构建面向他人的投顾适当性系统。

## 产品语义

- 原 `0-100` 综合分改称“观察分”。它表示当前规则下的关注优先级，不表示预期收益、上涨概率或买入建议。
- 五档输出改为：`high_attention`（高关注）、`attention`（较高关注）、`neutral`（中性观察）、`caution`（谨慎观察）、`low_attention`（低关注）。阈值沿用现有配置，保证结果可连续比较。
- 原“估值”统一改称“估值代理”，并展示具体代理方法。净值分位、指数价格分位和汇率分位不得表述为真实估值。
- 日报、图片和周报删除“强烈加仓”“可加仓”“闭眼买”“梭哈”等操作诱导文案。
- 所有报告明确说明：观察分不是收益预测或操作指令。

## 总体流程

```text
抓取数据
  -> 记录各数据源刷新状态
  -> 检查数据质量
  -> 计算技术/估值代理/事件分
  -> 计算观察分与可信状态
  -> 保存版本化审计快照
  -> 更新已到期的信号结果
  -> 生成日报或周报
```

评分组件继续保持纯函数边界。数据质量判断、分数组合和信号结果评估分别放在独立模块中，避免把抓取状态、评分规则和报告格式耦合在一起。

## 数据质量与降级

每次刷新记录数据源名称、基金或市场标识、尝试时间、最近成功时间、最新数据日期、写入行数和错误信息。

每只基金在评分前产生结构化质量结果：

- `reliable`：核心净值有效，三个评分维度均可用，所需市场数据未超过允许时效。
- `degraded`：核心净值有效，但持仓、市场、新闻或 LLM 中至少一项不可用或过期。
- `unscorable`：净值少于 60 条、缺少有效净值，或最新净值超过配置允许的最大时效。

默认时效配置采用宽松的自然日阈值以覆盖周末和节假日：净值 7 天、市场数据 7 天、持仓 180 天、新闻刷新 3 天。配置可覆盖这些值。

处理规则：

- `unscorable` 不生成数值观察分，但仍产生带问题列表的结果供报告展示。
- `degraded` 仅组合可用评分维度，并按这些维度的原权重重新归一化。报告必须紧邻观察分显示“降级”，并列出缺失项。
- 估值代理没有所需指数数据时可以使用现有净值分位回退，但必须记录实际方法并降低质量状态。
- LLM 调用失败、非法 JSON、字段类型错误、分数超出 `0-100` 或文本超长时，事件维度标记为不可用，不返回伪中性的 `50`。
- “刷新成功但没有相关新闻”是有效中性输入；“新闻刷新失败”是不可用输入，两者必须区分。

## 事件分析审计

事件结果增加分析状态、模型名称、生成时间和证据列表。证据列表保存输入给模型的新闻标题、来源、发布时间和 URL，以及结构化市场事件。报告最多展示最相关的少量证据，完整数据保存在评分快照中。

LLM 只总结已发生事件对当前观察优先级的影响，不给出加仓、减仓或未来价格预测。输出经过本地结构校验后才能进入观察分。

## 结果评估

结果评估只使用当时真实生成并保存的观察记录，不用未来数据重新构造历史事件分，也不为获得更多样本而回填虚构信号。

每条可评分记录在满 5、20、60 个基金交易日后计算：

- 基金区间收益率；
- 基准区间收益率；
- 超额收益率；
- 是否跑赢基准。

基准规则：

- `domestic_active` 和 `domestic_index` 使用沪深 300。
- `qdii_index` 使用人民币口径纳指基准，区间收益为 `(纳指末值 / 纳指初值) * (USD/CNY 末值 / USD/CNY 初值) - 1`。

期限按基金净值序列中的交易记录计数。基准取对应起止日期当日或其后的首个可用值，无法取得完整区间时暂不评价。

周报按评分版本、观察等级和期限汇总样本数、平均超额收益、中位超额收益和跑赢基准比例。任一分组样本少于 30 条时标注“证据不足”，不得据此自动调整权重或宣称信号有效。

## 数据存储与迁移

现有 `daily_scores` 保留以兼容历史部署，并通过幂等迁移增加：

- `observation_level`；
- `quality_status`；
- `quality_json`；
- `scoring_version`。

旧 `recommendation` 字段继续写入兼容值，但新代码和报告只读取 `observation_level`。旧记录在读取时映射为对应观察等级，并标记为旧评分版本，不重写其原始 JSON。

新增两张表：

- `data_source_status`：保存每个来源与标识的最近刷新状态。
- `signal_outcomes`：以基金代码、信号日期、评分版本和期限为唯一键，保存收益评价结果。

`init_db` 每次启动都执行可重复迁移，现有数据库不需要手工操作，也不删除旧数据。

## 报告设计

日报文字和图片展示：

- 观察分和观察等级；
- 数据可信状态和问题摘要；
- 技术、估值代理、事件三个分项；
- 数据截止日期和评分版本；
- 估值代理方法；
- 事件证据及 AI 分析状态；
- 非操作性风险说明。

存在 `unscorable` 基金时，报告保留该基金并显示不可评分原因，不用其他基金的结果掩盖数据故障。排序顺序为可评分基金按观察分降序，其后为不可评分基金。

周报保留观察分趋势，删除“可加仓信号次数”等描述，增加已到期的 5/20/60 日结果摘要。样本不足时展示累计样本量和最早预计可形成结论的状态，不展示空洞的胜率。

## 错误处理

- 单一基金抓取失败不阻断其他基金，但必须写入数据源状态并在该基金质量结果中体现。
- 新闻刷新失败不阻断技术和估值代理计算。
- 数据库迁移失败时终止任务，避免在未知表结构上继续写入。
- 信号结果计算失败只记录错误，不影响日报；下一次运行会重新尝试未完成期限。
- 报告图片失败继续降级为文字推送，沿用现有行为。

## 测试与验收

新增 `pytest` 测试，所有网络访问使用替身数据：

- 数据新鲜、过期、缺失、降级和不可评分判断；
- 可用维度权重重新归一化；
- LLM 非法 JSON、越界分数、错误字段、超长内容和接口异常；
- 5/20/60 日成熟判断和超额收益计算；
- 国内沪深 300 基准与 QDII 人民币口径基准；
- 数据库迁移重复执行且不丢失旧记录；
- 日报、周报和图片标签不包含操作诱导文案；
- 无网络的每日管道集成流程。

完成标准：

1. `pytest` 全部通过。
2. `python -m compileall -q src scripts` 通过。
3. 使用临时数据库和固定样本生成日报 Markdown 与 Dashboard 图片。
4. 人工检查图片无文字重叠，观察分、可信状态和免责声明清晰可见。
5. README 和配置示例准确描述观察工具定位、质量阈值及结果评估命令。

## 后续阶段

当同一评分版本在关键分组积累至少 30 条到期样本后，再审查各维度与超额收益的关系。只有样本外证据稳定，才进入权重调整、因子删减和真实 PE/PB 或持仓加权估值接入。任何模型升级必须提升独立验证集表现，而不是只改善历史回测曲线。
